## Catatan Pembelajaran
1.Single Responsibility Principle (SRP)
Saya memahami bahwa setiap class harus memiliki satu tanggung jawab utama. 
Jika satu class menangani terlalu banyak hal, maka perlu dilakukan refactoring 
agar tanggung jawab dipisah menjadi beberapa class yang lebih fokus.

2.Open/Closed Principle (OCP)
Saya mempelajari bahwa sebuah class harus “open for extension, closed for modification”. 
Artinya, ketika ada fitur baru, kita membuat class baru atau mengimplementasikan interface, 
bukan mengubah class lama yang sudah stabil.

3.Liskov Substitution Principle (LSP)
Saya belajar bahwa subclass harus bisa menggantikan peran superclass tanpa mengubah perilaku sistem. 
Setiap turunan harus tetap menghormati kontrak perilaku dari parent class.
